Movie API Project

CF Full-Stack Immersion Achievement 2

Application Objective:
To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

HTTP Requests and Responses:

Request Method URL Response
Return list of all movies. GET /movies res.json(movies)
Return data about specific movie GET /movies/:title res.json(movie)
Return data about a genre GET /movies/genre/:genreName res.json(genre)
Return data about a director GET /movies/director/:directorName res.json(director)
Register a new user POST /users res.json(user)
Update user info PUT /users/:username res.json(updatedUser)
User save movie to favorite list PUT /users/:username/movies/:movieID res.json(updatedUser)
User removes movies from favorite list DELETE /users/:username/movies/:movieID res.json(updatedUser)
User to deregister DELETE /users/:username confirmation message

Register a New User

  • Description:
    • Register a new user, by adding data to the users array
  • URL Endpoint: /users
  • Requests:
    • Method: POST
    • Query Parameters: n/a
    • Body: user data in JSON format
    • Example: 
                  {
              "username": "exampleJoe",
              "password": "examplePass",
              "email": "joe@example.com",
              "birthday": "10/29/90"
            }
  • Response:
    • Status: 201
    • Format: JSON
    • Example: 
                    {
                "username":"exampleJoe",
              "password":"examplePass",
              "email":"joe@example.com",
              "birthday":"1990-10-29T05:00:00.000Z",
              "favoritemovies":[],
              "_id":"659d93b0a502d2f4f192418d","__v":0
            }
